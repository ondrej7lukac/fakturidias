# Internationalization (i18n)

Source: `invoice-react/src/utils/i18n.ts`

## Supported Languages

| Code | Language |
|------|----------|
| `cs` | Czech (default) |
| `en` | English |

Slovak (`sk`) was removed from the language toggle. Do not re-add it without confirmation.

## How It Works

Language state lives in `App.tsx`:

```ts
const [lang, setLang] = useState<'cs' | 'en'>('cs')
```

`App.tsx` calls `getTranslations(lang)` from `src/utils/i18n.ts` to get a flat `Record<string, string>` object `t`, then passes both `lang` and `t` as props to every component that needs them.

## Component Contract

Every component that needs translated text accepts:

```ts
interface MyComponentProps {
  lang: string
  t: Record<string, string>
  // ... other props
}
```

Access translations as:
```tsx
<label>{t.invoiceNumber}</label>
<button>{t.save}</button>
```

## Adding a Translation Key

1. Open `invoice-react/src/utils/i18n.ts`
2. Add the key to **both** the `cs` and `en` objects
3. Use `t.yourNewKey` in the component

Never hardcode Czech or English strings in component files — always go through `t`.

## Language Toggle

The header renders a pill toggle (no dropdown):

```html
<div class="lang-toggle">
  <button class="lang-toggle__btn lang-toggle__btn--active">CS</button>
  <button class="lang-toggle__btn">EN</button>
</div>
```

Clicking a button calls `setLang('cs' | 'en')` from App, which cascades the new `t` object to all components.

## Date & Money Formatting

Language-aware formatting helpers live in `invoice-react/src/utils/storage.ts`:

```ts
money(value)         // Always 2 decimal places, e.g. "1 234,50"
formatDate(date)     // ISO string: "YYYY-MM-DD"
```

These are not locale-sensitive — formatting is fixed regardless of language.
