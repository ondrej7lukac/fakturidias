## Core Principles

### 1. Component-Scoped CSS Architecture

**Every component owns its CSS file.**

```
components/
├── Header.jsx          (component logic)
├── Header.css          (ONLY Header styles here)
├── Settings.jsx
├── Settings.css        ← All Settings styles in this file
├── InvoiceForm.jsx
└── InvoiceForm.css
```

**Benefits:**

- No global cascade pollution
- Styles auto-deleted when component deleted
- Easy to find where a style is defined
- No naming conflicts between components

**Rule:** If you add CSS, create a new `.css` file with the component name and import it:

```jsx
// Header.jsx
import './Header.css'; // ← Always add this import
```

### 2. CSS Variables for Theming

**No hardcoded colors in component CSS.**

Use CSS custom properties defined in `theme.css`:

```css
/* ❌ WRONG */
background: #0b1022;
color: #e9ecff;

/* ✅ RIGHT */
background: var(--color-bg);
color: var(--color-text);
```

**Why?** Light and dark modes are automatically applied by the theme system without editing component code.

**Common Variables:**

- `--color-bg` — Background
- `--color-text` — Primary text
- `--color-text-muted` — Secondary text
- `--color-surface` — Card backgrounds
- `--color-border` — Borders and dividers
- `--color-primary` — Main action color (green)
- `--color-danger`, `--color-warning` — Semantic colors

See **DESIGN.md → Color Palette** for complete list.

### 3. Responsive Mobile-First Design

**Code for mobile first, enhance for larger screens.**

```css
/* Base: Mobile (480px and up) */
.component {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

/* Tablet: 768px and up */
@media (min-width: 768px) {
  .component {
    flex-direction: row;
    gap: 2rem;
  }
}

/* Desktop: 1180px and up */
@media (min-width: 1180px) {
  .component {
    max-width: 1400px;
  }
}
```

**Test Breakpoints:**

- `480px` — iPhone/small phone portrait
- `768px` — iPad / large phone landscape
- `1180px` — Desktop / wide tablet

### 4. Dark Mode Support

**Automatic by design. No extra work needed.**

The app detects system preference:

```javascript
const isDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
```

And applies it to `<html data-theme="dark">` or `<html data-theme="light">`.

**Your job:** Use CSS variables (covered in #2 above). The theme system does the rest.

### 5. Accessibility is Built-In

**WCAG 2.1 AAA compliance by default.**

Checklist for every component:

- [ ] **Keyboard Navigation** — Tab through all controls, no traps
- [ ] **Focus Ring** — Visible 3px outline on focused elements (blue on light, green on dark)
- [ ] **Color Contrast** — Text is 4.5:1 ratio minimum (even in dark mode)
- [ ] **Labels** — Every `<input>` has a matching `<label for="id">`
- [ ] **Touch Targets** — Buttons ≥44×44px on mobile
- [ ] **Semantic HTML** — Use `<button>`, `<input>`, `<label>` correctly (not `<div>`)
- [ ] **ARIA** — Add `aria-label` for icon-only buttons, `aria-describedby` for error messages

See **DESIGN.md → Accessibility Standards** for details.

## Workflow: Component Creation

### Step 1: Define in DESIGN.md

Add the component spec to the component catalog (if not already there):

```markdown
### ComponentName

- **Purpose:** What does this component do?
- **Users:** Who interacts with it?
- **Key Behaviors:** Create/edit/delete/search/filter?
- **Layout:** Desktop-first or mobile-first?
```

### Step 2: Create JSX Component

```jsx
// components/MyComponent.jsx
import './MyComponent.css'; // ← Import at top

function MyComponent() {
  return (
    <div className='my-component'>
      {/* Use classNames scoped to this component */}
    </div>
  );
}

export default MyComponent;
```

### Step 3: Create Scoped CSS File

```css
/* components/MyComponent.css */

.my-component {
  background: var(--color-bg);
  color: var(--color-text);
  padding: 1.5rem;
  border-radius: 18px;
  border: 1px solid var(--color-border);
}

.my-component__item {
  display: flex;
  gap: 1rem;
  padding: 1rem;
}

.my-component__item:hover {
  background: var(--color-surface);
}

/* Responsive: Tablet and up */
@media (min-width: 768px) {
  .my-component {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 2rem;
  }
}

/* Responsive: Desktop and up */
@media (min-width: 1180px) {
  .my-component {
    grid-template-columns: 1fr 1fr 1fr;
  }
}
```

### Step 4: Test & Verify

- [ ] Light mode rendering
- [ ] Dark mode rendering (toggle in Settings)
- [ ] Mobile layout (DevTools 480px)
- [ ] Tablet layout (DevTools 768px)
- [ ] Desktop layout (DevTools 1180px)
- [ ] Keyboard tab navigation
- [ ] Focus indicators visible
- [ ] No console errors

### Step 5: Run Impeccable

```bash
# Review the design
npx impeccable audit MyComponent

# Check accessibility
npx impeccable audit MyComponent

# Polish before shipping
npx impeccable polish MyComponent
```

## Common Patterns

### Button Styling

```css
.my-component__button {
  padding: 0.75rem 1.5rem;
  background: var(--color-primary);
  color: #000;
  border: none;
  border-radius: 50px; /* Pill shape */
  font-weight: 700;
  cursor: pointer;
  transition: filter 0.15s ease;
}

.my-component__button:hover {
  filter: brightness(1.1);
}

.my-component__button:focus {
  outline: 3px solid var(--focus-ring);
  outline-offset: 2px;
}

.my-component__button:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}
```

### Form Input Styling

```css
.my-component__input {
  width: 100%;
  padding: 0.75rem 1rem;
  border: 1px solid var(--color-border);
  border-radius: 12px;
  background: var(--color-bg);
  color: var(--color-text);
  font-family: 'Plus Jakarta Sans', sans-serif;
  font-size: 1rem;
}

.my-component__input:focus {
  outline: none;
  border-color: var(--color-primary);
  box-shadow: 0 0 0 3px var(--focus-ring);
}

.my-component__input::placeholder {
  color: var(--color-text-subtle);
}
```

### Card/Panel Styling

```css
.my-component__card {

```

## CSS Naming Convention (BEM)

Use Block-Element-Modifier for clarity:

```css
/* Block: the component */
.my-component {
}

/* Element: child of block */
.my-component__header {
}
.my-component__item {
}
.my-component__button {
}

/* Modifier: variant or state */
.my-component__button--primary {
}
.my-component__button--danger {
}
.my-component__item--selected {
}
.my-component__item--disabled {
}
```

## Global CSS Rules

**Only truly global styles go in `index.css`:**

✅ **OK to put in index.css:**

- CSS reset (margin/padding resets)
- Body defaults (font-family, line-height)
- HTML baseline (scroll-behavior, box-sizing)
- Utility classes (.sr-only, .visually-hidden)
- System-wide defaults

❌ **NOT OK in index.css:**

- Component styles (goes in ComponentName.css)
- Layout for specific pages (goes in component file)
- State-dependent styling (goes in component file)

**The rule:** If you can name it as a component, it doesn't go in index.css.

## Performance Checklist

- [ ] **CSS is scoped** — No global cascade, each component owns its CSS
- [ ] **No unused styles** — Run `npx impeccable audit ComponentName`
- [ ] **Images optimized** — WebP with PNG fallback, max 2x resolution
- [ ] **Fonts lazy-loaded** — Plus Jakarta Sans uses `font-display: swap;`
- [ ] **Animations performant** — Use transforms + opacity, not layout properties
- [ ] **Layout shift minimized** — Reserve space for dynamic content
- [ ] **Bundle size checked** — `npm run build` and check bundle size

## Filing Bugs

When reporting CSS/design issues, include:

1. **Browser** (Chrome 120, Safari 17, Firefox 121, etc.)
2. **OS** (macOS 14, Windows 11, etc.)
3. **Theme** (Light/Dark)
4. **Screen size** (375px, 768px, 1920px)
5. **Component** (Header, Settings, InvoiceForm, etc.)
6. **Expected vs. Actual** (what should happen vs. what happens)
7. **Screenshot** (if visible issue)
